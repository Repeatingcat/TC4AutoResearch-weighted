# TC4AutoResearch

## 批量解题

将未完成的研究笔记放在玩家背包中，在研究台放好笔与墨，然后点击研究台右侧的 `批量解题`。程序会依次把笔记移入研究台、等待自动求解完成，再把已完成笔记移回背包，直到背包中没有未完成笔记。

按钮会显示 `批量 已完成数/总数`；再次点击可以手动停止。关闭研究台也会停止批量处理。批量模式只使用容器点击操作，不直接修改笔记 NBT。

遇到笔与墨耗尽、单张笔记 30 秒内未完成、容器同步失败或已完成笔记无法放回背包时，批量模式会停止并在聊天栏说明原因。运行期间不要手动移动研究台中的当前笔记。

## 要素权重

自动研究会计算每个候选解所消耗要素的总成本，并优先使用总成本最低的路径。成本越高，该要素越不容易出现在最终路径中；成本相同时，优先使用当前研究点更多的要素。

首次启动后，配置文件 `config/TCAutoResearch.cfg` 会生成 `[aspect_costs]` 分类。每个键是 Thaumcraft 要素的内部 tag，每个值是该要素的路径成本，例如：

```properties
I:aer=2
I:tempestas=512
I:custom1=1024
```

内置 GTNH 的 69 个常见要素成本。未列出的附属要素默认成本为 `16`，并会在游戏加载完要素注册表后自动加入配置。修改配置后重新进入游戏即可使用新的权重。

研究台右侧提供两个入口：

- `调试信息`：显示最近一次解题状态、耗时、加权总成本，以及每种已用要素的数量、单价和小计。这里显示的是该次实际传给求解器的权重快照。
- `要素权重`：搜索并分页编辑所有已注册要素的权重。点击保存后立即写入配置，下一次解题生效；恢复默认后仍需点击保存。

也可以使用客户端命令 `/TCAutoResearch debug` 和 `/TCAutoResearch weights` 打开对应窗口。

### 求解模式

- 只有两个固定要素时，使用带已占用格子状态的精确加权搜索。搜索完成后得到最低权重路径，调试窗口显示“精确”和“最优证明：是”。
- 三个及以上固定要素时，三种连接策略在统一的 500 毫秒窗口内收集候选，每条路径最多枚举权重排序后的前 256 个结果，去重后选择总权重最低的方案。调试窗口显示“混合”和候选数量；该模式不声明全局最优。
